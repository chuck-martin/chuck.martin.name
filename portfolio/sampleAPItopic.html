Sample API Topic

Working through Tom Johnson's Learn API Doc tutorial, an early page describes how to create a request for a REST endpoint. I got the inspiration that API docs are typically reference topics, so I thought I'd try to create a topic for this endpoint before I learned how to do it "properly." Part of the little epiphany was that an endpoint seems to be analogous to a function or a method, and then the keys are just parameters for the function/method. This is what I came up with:

API

https://simple-weather.p.mashape.com

Endpoint

weather

Returns current basic weather information for the defined longitude and latitude. The two parameter values together define a unique point on the Earth's surface.

Supported return types: text/plain

Parameters

lat

Required. Defines a latitude value. Latitude is a measurement of distance north or south from Earth's equator.

Valid values: Whole numbers in the range -90.0 to 90.0.

lng

Required. Defines a longitude value. Longitude is a measurement of distance east or west from a point known as the meridian, which is in Greenwich, England.

Valid values: Whole numbers in the range -180.0 to 180.0.

Limitations

Only certain ranges of latitude and longitude values are supported by this endpoint. Values out of the valid ranges or values that are not supported (typically, values that equate to points over the ocean and far away from weather reporting stations) will return an Unsupported response.

Example

The latitude and longitude coordinates for Santa Clara, California are lat: 37.3710062 and lng: -122.0375935.

https://simple-weather.p.mashape.com/weather?lat=37.3710062&lng=-122.0375935
