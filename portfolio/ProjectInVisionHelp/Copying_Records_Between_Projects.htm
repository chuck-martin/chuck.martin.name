Copying Records Between Projects

You can copy or move Issues, Documents, Defects, Risks, Changes, Requirements, or Custom Module records between Projects, whether those Projects are in the same or a different Workgroup. You can also create a Link between the two records. The only records that cannot be copied or moved are Resource and Journal records because of the direct relationship that exists between these and other records.

To copy a record between Projects

  1. In an Issues Documents, Defects, Risks, Changes, or Requirements, or Custom Module list view, select any record. (Do not click on a link field in the record.)
  2. Click  Tools, then select Transfer across Projects. The Transfer Across Projects dialog box appears.
  3. Select one or more records in the From list.
    Hold down the Shift key while clicking to select a contiguous group of records.
    Hold down the Ctrl key while clicking to select multiple discrete records.
  4. In the Action box, select Copy or Move.
    • Select Copy to make a copy of the record(s) in the new module. Select Create Link to add a link from the original record to the new record, Project, or both.
    • Select Move to move the record(s) to the new module.
  5. In the Workgroup and Project drop-down lists in the To box, select the Workgroup and Project where you want to copy the record.
    You can select only from Workgroups you're authorized to access.
  6. Click Add.
    You can also click Add All to copy all the records.
  7. Click Copy or Move.
