Copying Records Within Projects

You can copy or move Issues, Defects, Risks, Changes, and Requirements records within a Project, between Project modules. You can also create a link between the two records.

To copy a record within a Project

  1. In an Issues Defects, Risks, Changes, or Requirements module list view, select any record. (Do not click on a link field in the record.)
  2. Click  Tools, then select Transfer across Modules. The Transfer Across Modules dialog box appears.
  3. Select one or more records in the From list.
    Hold down the Shift key while clicking to select a contiguous group of records.
    Hold down the Ctrl key while clicking to select multiple discrete records.
  4. In the Action box, select Copy or Move.
    • Select Copy to make a copy of the record(s) in the new module. Select Link to add a link from the original record to the copy.
    • Select Move to move the record(s) to the new module.
  5. In the Module drop-down list in the To box, select the module where you want to copy the record.
  6. Click Add.
    You can also click Add All to copy all the records.
  7. Click Copy or Move.

When records are copied across modules, values are copied from one specific field to another specific field. For example, if Defect records are copied to the Issues module, the Defect # would be copied to Issue #, Defect Title to Issue Title, Defect Priority to Issue Priority, etc. These mappings are not configurable.
