Quick Filters

Project InVision allows you to generate single-condition Quick Filters in most lists of records (not reports or charts) without the need to create or save advanced Filters. The Quick Filter function is used to display only those records that meet the Quick Filter criteria. In addition, Quick Filters can be built on top of other Quick Filters, which allows you to drill down in the record set to get more specific information.

To create a Quick Filter

  1. View a module list where the records are to be filtered.
  2. Right-click a field to be queried and select Create Quick Filter. The Quick Filter dialog box appears with the Field name in a read-only format.
  3. In Test, select an operator. You can choose from:
    < Less than the entered value.
    <= Less than or equal to the entered value.
    = Equal to the entered value.
    > Greater than the entered value.
    >= Greater than or equal to the entered value.
    Not Not the entered value.
    Starts With Starts with the entered text.
    Contains Contains the entered text.
    Ends With Ends with the entered text.
    Doesn't Start With Does not start with the entered text
    Doesn't Contain Does not contain the entered text.
    Doesn't End With Does not end with the entered text.
    Sounds Like Close phonetically. (For example, “Smith” would find “Smythe” and perhaps others.)
    Is Null Contains nothing (is blank).
    Is Not Null Contains something (is not blank).
  4. In Filter Value, enter the text that the filter will use. (The default is the full content of the selected field.)
  5. Select a Quick Filter Option:
    • Select Create New Filter to make this the only filter used.
    • Select Append to Current Filter <current filter name> to apply this filter on top of the current filter.
  6. Click Submit.
