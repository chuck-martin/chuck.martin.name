Documents Detail Form

This form contains information about a Document record, including the type of document, the author, its Status, last revision date, when it is due, the original template used, the document version, and the document file association. The form contains several pages, each containing a specific type of information. Click the links under Pages to view each page. The form also has its own toolbar that allows quick access to common functions.

Documents Detail Form Pages 

These are the default pages. Your Workgroup Administrator may have renamed existing pages or added new ones, or may have renamed, added, or deleted fields within any page.

Information

This page shows Document-related information, such as the description of the Document record, its Revision and Due Dates, and Version.

  • Document # is generated by Project InVision and is not editable. It is one way to identify the Document.
  • Document Title is a brief (up to 40 characters) description of the Document. This is what displays in Project InVision and is not the title taken from the associated document file.
  • Type provides a category for the Document.
  • Author identifies which Project InVision user created the original document.
  • Revision Date is the date that the associated document was last revised and checked in.
  • Due Date is the date by which work on the associated document should be completed.
  • Workflow defines the order in which the Status of a particular record may be changed in Project InVision. Each time the Status of a Project InVision record is changed, Project InVision logs the date, time, and User responsible for the change. Click the Status label to see Status History.
    The Workgroup Administrator defines the Workflows that are available for each individual Workgroup.
  • Status defines steps within a workflow.
  • Status Revised is an uneditable field that shows who updated the Issue to the current status and when.
  • Template is uneditable information that identifies what template was used to create the Document record (not any associated document file).
  • Version shows the latest version of the checked in file.
  • Association shows the filename of the associated document file.
  • Checked In/Checked Out shows the date and time that the associated document file was checked in or checked out and who performed the action.

Comments

This page is for free-form comments that are maintained with the Document record. See Detail Form Common Pages, Comments.

Versions

This page shows the versions of the document files that have been checked in. You can view any version and mark a version to be kept.

Approvals

This page shows who is responsible for approving the Document record. Approvers may be defined automatically as a part of a Workflow or may be added manually. If approval is required as part of a Workflow, once the Document record reaches a pre-defined Status, it must be approved before it can continue to the next Status.

Keywords

This page allows you to define keywords for the Document. The keywords you can choose are defined by the Workgroup Administrator. See Detail Form Common Pages, Keywords.

Links

This page contains links to other records within Project InVision or to web addresses outside Project InVision. Links within Project InVision help you track associations between Projects and other items, such as Tasks and Resources. See Detail Form Common Pages, Links.

Discussions

This page contains threaded Discussions related to the Document. You can create a Discussion item, respond to the sender of a Discussion item, or respond to the group of users that are involved in the Discussion. See Detail Form Common Pages, Discussions.
