Creating Resources

Best practices are that the Workgroup Administrator should maintain Resources at the Enterprise level and add Resources to the appropriate Workgroups. A Resource may belong to one or more Workgroups. Only users with proper security may add Resources.

Resources may be assigned to multiple Workgroups, each Workgroup with its own customizations and configurations, such as workweek configuration, Non-Project Activities, Cost Centers, and Billing Codes.

  • Select a Primary Workgroup to use those customizations and configurations in all Workgroups where the Resource might be assigned.
  • Leave the Primary Workgroup blank to have the Workgroup-specific configurations and customizations apply to the Resource when assigned to the Workgroup.

Creating Resources can be done only by users with access to Administrative functions.

To create a Resource

  1. Click Administration. The Project InVision Administration window appears.
    The link is in the upper-right part of the window. If you do not have System Administrator or Workgroup Administrator access, you will not see this link.
  2. Under Select Screen, click Resources.
  3. Click  New Resource. The New Resources window appears.
  4. Enter the Resource information. Fields with bold labels are required. Click the links under Pages to enter more information.
  5. Click Submit.

If you want the Resource to use Project InVision, you must also create a Resource login.
