Defects Module Views

Default Views have been created for the Defect module. Your security access determines what Views you can see.

Views may be customized by the Workgroup Administrator or a Report Writer, so some View names or fields may not exist or may be labeled differently than shown here.

-Standard-

This View shows almost all the information about Defects. Fields: Defect Number, Defect Title, Cause, Status, Modified by, Date Modified, Submitted by, Priority, Discovery Date, Completion Date, Type, Severity, Evaluation Date, Date Entered, Entered by, Project Number, Project Name, Comments, Description, and Resolution.

Actual Work Pie

This View is a Pie Chart with slices based on Rollup selection and the Date Range specified. Depending on the Rollup used, the Work Hours will display for the specified Date Range, grouped by Rollup. Depending on the rollup used, the Work hours display for the specified Date Range, grouped by Rollup. Numbers can be rolled up by Type, Priority, Severity, Status, Cause, Responsible, Submitted by, or Project. Fields: Defect Number, Defect Title, Status, Actual Work, and Timesheet Date.

Count Chart

This View shows a total count of Defects, separated into groups by Rollup. Numbers can be rolled up by Type, Priority, Severity, Status, Cause, Responsible, Submitted by, or Project.

Defect Count Pie

This View is a Pie Chart with slices based on Rollup selection and the Date Range specified. Depending on the Rollup used, the Work Hours will display for the specified Date Range, grouped by Rollup. Numbers can be rolled up by Type, Priority, Severity, Status, Cause, Responsible, Submitted by, or Project.

Defect Matrix

This View shows a matrix that compares Defect Severity to Defect Priority. It separates the counts into High, Medium, and Low Severity versus High, Medium, and Low Priorities.

Defect Trend Chart

This View shows how Defects have progressed over time.

Personal Count Chart

This View shows a total count of Defects by Rollup and the number of Defects per Rollup group where the current user is the Responsible party. Numbers can be rolled up by Type, Priority, Severity, Status, Cause, Responsible, Submitted by, or Project.
