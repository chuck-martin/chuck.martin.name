Copying Projects Between Workgroups

You can copy or move Projects between Workgroups that you can access. When a Project is copied or moved, all associated records are copied or moved with it: Project schedule associations and files, Tasks, Issues, Document records and files, Defects, Risks, Changes, Requirements, Journals, and Custom module records. You can also create links between the two records.

To copy a Project to another Workgroup

  1. In a Projects list view, select any record. (Do not click on a link field in the record.)
  2. Click  Tools, then select Transfer across Workgroups. The Transfer Across Workgroups dialog box appears.
  3. Select one or more records in the From list.
    Hold down the Shift key while clicking to select a contiguous group of records.
    Hold down the Ctrl key while clicking to select multiple discrete records.
  4. In the Action box, select Copy or Move.
    • Select Copy to make a copy of the Project(s) in the new Workgroup. Select Link to add a link from the original Project to the copy.
    • Select Move to move the Project(s) to the new Workgroup.
  5. In the Workgroup drop-down list in the To box, select the Workgroup where you want to copy the Project.
    You can select only from Workgroups you're authorized to access.
  6. Optionally select what to include with the copy.
    • Select Pull-down Lists to include custom pull-down lists that have been created for the Project(s).
    • Select MPP File to include associated project schedule files.
  7. Click Add.
    You can also click Add All to copy all the Projects.
  8. Click Copy or Move.

Timesheet records are moved only when a Project record is moved, not when it is copied.

Project schedule files are automatically transferred when a Project is moved. If the Project is being copied, you have the option of also copying a version of the Project file.

If Pull-down List values are not copied and do not exist in the other Workgroup, the fields appear with blank or NULL values.
