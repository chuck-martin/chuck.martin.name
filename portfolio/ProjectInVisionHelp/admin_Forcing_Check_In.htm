Forcing Check In

When users lose checked out files that are associated with records or are just not available to check them in, you can force check in the file. Forcing check in nullifies the originally checked out file and makes the most recently checked in version the current version.

Forcing check in can be done only by users with access to Administrative functions.

To force check in of a document

  1. Click Administration. The Project InVision Administration window appears.
    The link is in the upper-right part of the window. If you do not have System Administrator or Workgroup Administrator access, you will not see this link.
  2. Under Select Screen, click Force Check-in.
  3. In the Navigation menu, select a Workgroup and a Module. All the currently checked out files are listed.
  4. Click the File Name of a document to select it.
    Do not click in the field that contains the link.
  5. Click  Force Check In.
  6. Click Yes.

A Notification is sent to the Resource who had the file checked out, notifying them that their version is no longer the master version of the file.
