Issues Module Views

Default Views have been created for the Issues module. Your security access determines what Views you can see.

Reports may be customized by the Workgroup Administrator or a Report Writer, so some View names or fields may not exist or may be labeled differently than shown here.

-Standard-

This View shows almost all the information entered for Issues in the Detail Form. Fields: Issue Number, Issue Title, Responsible, Submitted by, Date Opened, Date Due, Date Completed, Impact, Status, Priority, Project Number, Project Title, Comments, Description, and Resolution.

Actual Work Pie

This View is a Pie Chart with slices based on Rollup selection and the Date Range specified. Depending on the Rollup used, the Work Hours will display for the specified Date Range, grouped by Rollup. The Rolled up can be Type, Priority, Severity, Status, Responsible, Submitted by, or Project. Fields: Issue Number, Issue Title, Status, Actual Work, and Timesheet Date.

Count Chart

This View shows a total count of Issues by Rollup. Numbers can be rolled up by Type, Priority, Severity, Status, Responsible, Submitted by, or Project.

Detail

This View shows detailed information about almost all the information entered for Issues in the Detail Form. Fields: Issue Number, Issue Title, Responsible, Submitted by, Description, Resolution, Date Opened, Date Due, Date Completed, Priority, Status, Impact, Project Number, and Project Title.

Issue Count Pie

This View is a Pie Chart with slices based on Rollup. Depending on the rollup used, the counts will display grouped by the various values that exist for the Rollup field. Numbers can be rolled up by Type, Priority, Severity, Status, Responsible, Submitted by, or Project. Fields: Issue Number, Issue Title, Status, and Count.

Issue Matrix

This View shows a matrix that compares Issue Severity to Issue Priority. It separates the counts into High, Medium, and Low Severity versus High, Medium, and Low Priorities.

Issue Trend Chart

This View shows a graphical View of how Issues have progressed over time.

Issues

This View shows information about when Issues are due, what their Status and Priority are, and who is responsible for the Issue. Fields: Issue Number, Issue Title, Submitted by, Responsible, Date Opened, Date Due, Status, Priority, Project Number, and Project Title.

Personal Count Chart

This View gives a total count of Issues, separated into groups by Rollup and showing the number of Issues per Rollup group where the current user is the Responsible party. Numbers can be rolled up by Type, Priority, Severity, Status, Responsible, Submitted by, or Project.

Responsibility

This View shows information about who is responsible for selected Issues: the responsible party, the description, the Priority, and the Status of Issue information related to the Project. Fields: Issue Number, Issue Title, Description, Responsible, Submitted by, Status, Priority, Project Number, and Project Title.

Urgency

This View shows information about the urgency of selected Issues: the Issue, its Priority, Status, and Impact, and when the Issue is due. Fields: Issue Number, Issue Title, Status, Priority, Impact, Date Due, Date Completed, Responsible, Submitted by, Description, Project Number, and Project Title.
