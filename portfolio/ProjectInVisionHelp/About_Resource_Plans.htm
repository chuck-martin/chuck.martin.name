About Resource Plans

Resource Plans allow managers to quickly map the resources that will be required throughout the life of a Project. Requirements can be entered directly into the Resource Plan form, or the Resource Plan can be automatically updated based on the project schedule. See Updating Resource Plans.

The resource requirements defined in Resource plans can be used as the basis to calculate project costs (see Managing Project Costs) or to make educated decisions on how resources should be best assigned, and how projects should be prioritized within the portfolio (see Resource Analysis Reports).

You can create multiple Resource Plans for a single Project, but only one can be the “Current” Resource Plan and only one can be the “Baseline” Resource Plan. The plan marked as “Current” will be used for Resource Analysis Reports. By toggling between multiple resource plans, managers can analyze different “What-If” scenarios.

For example, a manager might define two Resource Plans for a single Project: the first identifies resource requirements if the project starts on January 1, the second identifies resource requirements if the project starts on July 1. If the January 1 plan is set as the “Current” Resource Plan, Resource Analysis Reports will analyze availability for this time frame. If the July 1 Resource Plan is set as the “Current” Resource Plan, Resource Analysis Reports will provide a new interpretation based on the new time frame. These two scenarios can be compared to enable an educated decision, based on a true analysis of overall resource availability, to move forward with the project as of January 1 or July 1.
