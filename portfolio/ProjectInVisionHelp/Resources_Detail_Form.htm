Resources Detail Form

The Resources Detail Form contains information about Resources, including Name, Title, Department, Region, Manager, skills, availability, contact information, etc. The form contains several pages, each containing a specific type of information. Click the links under Pages to view each page. The form also has its own toolbar that allows quick access to common functions.

Resources Detail Form Pages 

These are the default pages. Your Workgroup Administrator may have renamed existing pages or added new ones, or may have renamed, added, or deleted fields within any page.

Information

This page shows editable Resource information: the Resources Number, First Name, Last Name, Title, Region, Manager, Department, Billing Rate, Internal Rate, Billing code, and Cost Center. You can also see the User Login, which is not editable. You can mark the Resource as Active or Inactive (inactive Resources are not available to assign to Projects or Tasks) or as a Generic Resource (Generic Resources have no First Name or UserID and are used for planning).

  • First Name is the first or given name of the Resource and is available only when the Generic check box is cleared.
  • MI is the first letter of the Resource's middle name and is available only when the Generic check box is cleared.
  • Last Name/Name is the last or family name of the Resource when the Generic check box is cleared or the name given to a generic Resource when the Generic check box is selected.
  • Resource Title is your company's title for the Resource's position within your company.
  • Department is the department where the Resource is assigned, used to categorize and search for Resource records.
  • Region is the region where the Resource is located, used to categorize and search for Resource records.
  • Billing Code identifies what category Resource costs belong to.
  • Cost Center identifies what department is responsible for paying Resource costs.
  • Skill Category, defines skills that the Resource has and is used for searching and in defining what Resources are appropriate and available for Projects.
  • Level identifies the experience level of the Resource and is used for searching and in defining what Resources are appropriate and available for Projects.
  • Manager is the Resource's direct report and defines where information and documents go in some workflows.
  • Primary Workgroup is the Project InVision Workgroup that the Resource enters by default when logging in.
  • Billing Rate is the amount charged per hour to external clients.
  • Internal Rate is the amount charged per hour to internal clients.
  • Active defines whether the Resource is available to work on Projects.
  • Generic defines if the Resource represents a real person (the check box is cleared) or a template that can be added to a project that will later be filled with a real person.

Contact Info

This page contains Resource contact information such as telephone numbers, email addresses, and home addresses. You can store multiple phone numbers and Email addresses in this page. See Editing Contact Information.

Comments

This page is for free-form comments that are maintained with the Resource record. See Detail Forms: Common Pages, Comments.

Assignments

This page contains an uneditable hierarchical view of the Projects, Tasks, and Task-related items (Issues, Defects, Risks, Changes, and Requirements) that a Resource has been assigned to.

Skills

This page shows Resource skills. This information is used to filter Resource records to find people who have certain skills or sets of skills. You can store multiple skills in this page. See Editing Skills.

The Workgroup Administrator configures the Skill values based on the needs of the Workgroup.

Availability

This page is where you identify when you are working on Non-Project Activities and are unavailable for Task assignments, and when you are unavailable (or available) for only part of the work week. See Editing Availability.

The Availability Profile and Non-Project Activity information is related to Resource Planning. When Resource Plans are developed, Analysis Reports can compare the Resource requirements against availability of Resources. Managers can then see if a specific Resource can be assigned to a Project or if that Resource is over-allocated for the time period. For more information on Resource Planning, see Understanding Resource Plans.

Keywords

This page contains Keywords, which are values defined by the Workgroup Administrator that can be used as filters when searching. See Detail Forms: Common Pages, Keywords.

Links

This page contains links to other records within Project InVision or to web addresses outside Project InVision. Links within Project InVision help you track associations between Projects and other items, such as Tasks and Resources. See Detail Forms: Common Pages, Links.
