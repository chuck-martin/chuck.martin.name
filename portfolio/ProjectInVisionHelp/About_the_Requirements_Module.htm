About the Requirements Module

The Project InVision Requirements Module is where you manage and collaborate on project or product-related Requirements as they arise. A Requirement is a proposed feature or enhancement to a product (such as a feature request in a software product), defined as a Project is being scoped. In the Requirements Module, you track Requirements from request through evaluation to implementation. You can view Requirement information in list views or in the Requirements Detail Form.

The Project InVision Requirements module captures detailed information about Requirements, including the Name, Description, Priority, Status, Workflow, Submitted by Resource, and Responsible Resource.
