Advanced Filters

Advanced filters allow you to apply multiple criteria against records to get specific information results. Security profiles determine your ability to create and edit filters.

To apply a filter without specifying multiple criteria, you can apply a Quick Filter on any field in the current View.

To create an advanced filter

  1. In the Navigation menu, click Filter (not the down arrow next to the word). The Filters dialog box appears.
  2. Click New Filter.
    If you want to extend an existing filter, select it from the Based on drop-down list.
  3. In the Name field, enter a name of up to 25 characters.
  4. Enter the details for one or more criteria:
    1. In List Name, select the list that contains the field to be used in the filter.
      Complex Filters may be based on a maximum of two lists per Filter. One of the two lists must be the current list.
    2. In Field Name, select a field. The choices you see are based on the selection in List Name.
    3. In Test, select an operator. You can choose from:
      < Less than the entered value.
      <= Less than or equal to the entered value.
      = Equal to the entered value.
      > Greater than the entered value.
      >= Greater than or equal to the entered value.
      Not Not the entered value.
      Starts With Starts with the entered text.
      Contains Contains the entered text.
      Ends With Ends with the entered text.
      Doesn't Start With Does not start with the entered text
      Doesn't Contain Does not contain the entered text.
      Doesn't End With Does not end with the entered text.
      Sounds Like Close phonetically. (For example, “Smith” would find “Smythe” and perhaps others.)
      Is Null Contains nothing (is blank).
      Is Not Null Contains something (is not blank).
    4. In Value, enter or select a qualifier. Click in the field and type in a value, or click in the field and then click the ellipsis button to select from the Select Values dialog box. The choices you see are based on the selection in Field Name.
      You can have multiple items in the Value field by typing values, separated by semicolons, or by selecting multiple values in the Select Values dialog box.
      Custom Fields may also be selected by choosing the “ModuleName Custom” item (i.e., Projects Custom, Resources Custom, etc.). Calculated Fields may also be selected by choosing the “ModuleName Calculated” item (i.e. Projects Calculated, Resources Calculated, etc.).
    5. To use multiple criteria, click to select the appropriate And/Or operator.
      The And operator creates conditions where both the first condition and the second condition must be met. The Or operator creates more inclusive conditions because either the first condition or the second condition can be met.
      Click on the bottom row to add a row below, or click on a row and then click Insert Filter Row to add a row directly above. Then repeat steps a through d for each Filter criteria.
    6. If you define multiple criteria, you can click on the Parenthesis field on the end of either row to create additional logic in the Filter. The parentheses in a Filter serve the same purpose as they would in a mathematical equation. Elements in parentheses are evaluated first.
  5. In the Sort By list, select a Field Name to sort by, then double click on the Asc/Desc field to select ascending or descending order. Select a Field Name in the bottom row to add additional rows of sort criteria.
  6. Click Submit.

Filter Functions

  • Current User() - This will filter a list for the Current User logged into the Project InVision system. This Filter criterion is usable only for a Resource field in any list. One example of this Filter is where a Filter is needed at the Projects module level to show “My Projects.” Instead of creating a “My Projects” Filter for every single User Login, only one Filter needs to be created where “Manager = CurrentUser()” and each time the Filter is applied, it looks to the current logged-in user and runs a Filter for that person where the Manager field is equal to the Current User's Resource ID.
  • ResRollupTo(ResourceName) - This rolls up the criteria to the selected Resource Name. This Filter criterion is currently usable only on the Manager field in the Resources module. This Filter would be used when you want to filter all of the resource records to display only those resources whose Manager = a specified Resource.
  • Current Date(FieldName) - This filters a list for the Current Date. This Filter criterion is usable only on Date fields. One example of this Filter is when a Filter is needed to show all tasks that are due to start today. A Filter would be created at the Tasks module level where Current Start = Current Date (The current date used will be the machine date of the workstation).
  • Field(FieldName) - This filters a list for the specified field name. This Filter criterion is usable only for a Resource field in any list. One example is a Filter to show all Projects where you are both the Manager and the Sponsor of a Project record. A Filter would be created where Manager = Field(Sponsor).
  • ProjRollupTo(ProjectNumber) - This rolls up the criteria to the selected Project Number. This Filter criterion is usable only for a Projects list. This Filter can be used to filter a Projects list to show only the Projects for a specified parent Project. A Filter would be created where Parent Project = ProjRollupTo(Project Number).

Notes on Filters

Modules

  • Filters may be based on a maximum of two Modules per Filter. One of the two Modules MUST be the current Module. Custom fields may also be selected by choosing the “ModuleName- Custom” item (i.e., Projects Custom, Resources Custom, etc.).
  • “ModuleName Custom” entries will appear only if Custom Fields have been defined for the module in the specific Workgroup in which the Filter is being created.

Field Names

  • Field Names appear based on the module that is chosen. If a standard module is chosen, then all standard fields for that module are displayed in the Field Names pull-down. If a Custom Module is chosen (i.e., Resources Custom), then only the Custom Fields for that module appear in the Field Names pull-down.
  • If no Custom Fields have been created for a particular module, then the “ModuleName Custom” entry does not appear in the Module pull-down.

Test

The Tests are expressions that are run against the data to see if they meet a specific value. The expression is always created in terms of Field, Test, and Value (i.e., Project Number = 10). If the expression is true, then the data is presented.

Values

The Value can be typed directly into the Value field of the Filters dialog box, unless an ellipsis appears. If an ellipsis appears, you must click on the ellipsis to open the Values List dialog box.
