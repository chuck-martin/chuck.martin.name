Issues Detail Form

This form contains information about an Issue as it is tracked from discovery through evaluation to resolution. The form includes several pages, each containing a specific type of information. Click the links under Pages to view each page. The form also has its own toolbar that allows quick access to common functions.

Issues Detail Form Pages 

These are the default pages. Your Workgroup Administrator may have renamed existing pages or added new ones, or may have renamed, added, or deleted fields within any page.

Information

This page shows Issues-related information, such as the description of the Issue, its impact, any Resolution, the Type of Issue, its Priority and Severity, and which Resources are involved in the Issue. Dates are used to track when the Issue was Opened, Due, and Closed.

  • Issue # is generated by Project InVision and is not editable. It is one way to identify the Issue.
  • Issue Title is a brief (up to 40 characters) description of the Issue.
  • Description can contain as much text as necessary; enter and add details as the Issue progresses through the workflow.
  • Impact is a brief (up to 255 characters) description of potential consequences that could occur if the Issue is not resolved or of what other items are affected by the existence of this Issue.
  • Resolution can contain as much text as necessary to document the Issue resolution.
  • Workflow defines the order in which the Status of a particular record may be changed in Project InVision. Each time the Status of a Project InVision record is changed, Project InVision logs the date, time, and User responsible for the change. Click the Status label to see Status History.
    The Workgroup Administrator defines the Workflows that are available for each individual Workgroup.
  • Status defines steps within a workflow.
  • Status Revised is an uneditable field that shows who updated the Issue to the current status and when.
  • Type is the type of the Issue, used to categorize and search for Issue records.
  • Priority is the priority of the Issue, used to categorize and search for Issue records.
  • Severity is the severity of the Issue, used to categorize and search for Issue records.
  • Submitted By is the Resource who raised or entered the Issue.
  • Responsible is the Resource who is currently responsible for the Issue at this status in the workflow.
  • Task Name is the task that the Issue is associated with. If an Issue is associated with a specific Task, then the Issue is displayed in the Time Tracking application so that time can be posted against the Issue and rolled up into the Project Task. See Time Tracking.
  • Date Opened is the date that the Issue was first raised (which may be before it was actually entered into Project InVision.
  • Date Due is the date the Issue is scheduled to be resolved.
  • Date Completed is the date the Issue was actually resolved.

Comments

This page is for free-form comments that are maintained with the Issue record. See Detail Form Common Pages, Comments.

Keywords

This page allows you to define keywords for the Issue. The keywords you can choose are defined by the Workgroup Administrator. See Detail Form Common Pages, Keywords.

Links

This page contains links to other records within Project InVision or to web addresses outside Project InVision. Links within Project InVision help you track associations between Projects and other items, such as Tasks and Resources. See Detail Form Common Pages, Links.

Discussions

This page contains threaded Discussions related to the Issue. You can create a Discussion item, respond to the sender of a Discussion item, or respond to the group of users that are involved in the Discussion. See Detail Form Common Pages, Discussions.
