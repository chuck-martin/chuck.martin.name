Defects Detail Form

This form contains information about Project Defects related to tracking the Defect from discovery through evaluation to resolution. The form contains several pages, each containing a specific type of information. Click the links under Pages to view each page. The form also has its own toolbar that allows quick access to common functions.

Defects Detail Form Pages 

These are the default pages. Your Workgroup Administrator may have renamed existing pages or added new ones, or may have renamed, added, or deleted fields within any page.

Information

This page shows Defect-related information, such as the description of the Defect, its cause, any Resolution, the Type of Defect, its Priority and Severity, and which Resources are involved in the Defect. Dates are used to track when the Defect was Opened, Evaluated, and Closed.

  • Defect # is generated by Project InVision and is not editable. It is one way to identify the defect.
  • Defect Title is a brief (up to 40 characters) description of the defect.
  • Description can contain as much text as necessary; enter and add details as the defect progresses through the workflow.
  • Cause is a brief (up to 255 characters) description of what caused the defect.
  • Resolution can contain as much text as necessary to document the defect resolution.
  • Workflow defines the order in which the Status of a particular record may be changed in Project InVision. Each time the Status of a Project InVision record is changed, Project InVision logs the date, time, and User responsible for the change. Click the Status label to see Status History.
    The Workgroup Administrator defines the Workflows that are available for each individual Workgroup.
  • Status defines steps within a workflow.
  • Status Revised is an uneditable filed that shows who updated the defect to the current status and when.
  • Type is the type of the Defect, used to categorize and search for Defect records.
  • Priority is the priority of the Defect, used to categorize and search for Defect records.
  • Severity is the severity of the Defect, used to categorize and search for Defect records.
  • Submitted By is the Resource who found or entered the defect.
  • Responsible is the Resource who is currently responsible for the defect at this status in the workflow.
  • Task Name is the task that the defect is associated with. If a Defect is associated with a specific Task, then the Defect is displayed in the Time Tracking application so that time can be posted against the Defect and rolled up into the Project Task. See Time Tracking.
  • Discovery Date is the date that the defect was first found (which may be before it was actually entered into Project InVision.
  • Evaluation Date is the date the defect was evaluated (for impact, resource use, and planning).
  • Completion Date is the date the defect was actually resolved.

Comments

This page is for free-form comments that are maintained with the Defect record. See Detail Forms: Common Pages, Comments.

Keywords

This page allows you to define keywords for the Defect. The keywords you can choose are defined by the Workgroup Administrator. See Detail Forms: Common Pages, Keywords.

Links

This page contains links to other records within Project InVision or to web addresses outside Project InVision. Links within Project InVision help you track associations between Projects and other items, such as Tasks and Resources. See Detail Forms: Common Pages, Links.

Discussions

This page contains threaded Discussions related to the Defect. You can create a Discussion item, respond to the sender of a Discussion item, or respond to the group of users that are involved in the Discussion. See Detail Forms: Common Pages, Discussions.
