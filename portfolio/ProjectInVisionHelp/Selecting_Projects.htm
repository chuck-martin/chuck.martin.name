Selecting Projects

Because most records, such as Tasks, Issues, Defects, and Risks, are associated with Projects, the records you see in a module depend on the Project selected. (Filters and Groupings control how many records you see and how they are organized.) You can select Projects in the Selector, in a Projects module list view, or by using the Find feature.

To select Projects in the Selector

In the Selector, you can select one or more projects.

  1. If the Selector hierarchy view is not visible, click  Selector.
  2. Click a Project name to select a single Project.
    -or-
    Select the check boxes for Projects to select multiple discrete Projects.
    -or-
    Click a hierarchy level to select all the Projects in that group.
    -or-
    Click -All- to select all Projects.
  3. Click Go.
    The selected Project or Projects will appear in the list view on the right.

If you're in the Project module, you'll see the Project(s) listed. If you're in any other module, you'll see all the records associated with the selected Project(s). If you've selected one Project, its name will appear in the Project Indicator. If you've selected an entire hierarchy level, the text “Level:,” followed by the hierarchy level name, will appear in the Project Indicator. If you've selected multiple discrete Projects, “Multiple selections” will appear in the Project Indicator.

To select a Project in a Project module list view

In a Project module list view, you can select a single Project.

  1. Under Modules, click Projects.
  2. In a Projects list view, click a Project record. The Project is highlighted in yellow and appears in the Project Indicator.

Until you select a different project, the records you see when you work in other modules will be the records associated with that Project.

To select a Project using Find

You can select a Project by using the Find feature.

  1. Click  Find next to the Project Indicator. The Find dialog box appears.
  2. Find and select a Project, then click Select.
