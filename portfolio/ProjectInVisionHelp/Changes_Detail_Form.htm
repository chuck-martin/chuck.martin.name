Changes Detail Form

This form contains information about a Change request, information related to tracking the Change from definition through evaluation to resolution. The form contains several pages, each containing a specific type of information. Click the links under Pages to view each page. The form also has its own toolbar that allows quick access to common functions.

Changes Detail Form Pages

These are the default pages. Your Workgroup Administrator may have renamed existing pages or added new ones, or may have renamed, added, or deleted fields within any page.

Information

This page shows Change-related information, such as the description of the Change, its impact, the Business Case behind it, the Type of Change, its Priority and Complexity, and which Resources might be involved in implementing the Change. Dates are used to track when the Change was Evaluated and Closed.

  • Change # is generated by Project InVision and is not editable. It is one way to identify the Change.
  • Change Title is a brief (up to 40 characters) description of the Change.
  • Description can contain as much text as necessary; enter and add details as the Change progresses through the workflow.
  • Impact is a brief (up to 255 characters) description of potential consequences that could occur if the Change is implemented.
  • Business Case can contain as much text as necessary to document the business reasons for making the change.
  • Workflow defines the order in which the Status of a particular record may be changed in Project InVision. Each time the Status of a Project InVision record is changed, Project InVision logs the date, time, and User responsible for the change. Click the Status label to see Status History.
    The Workgroup Administrator defines the Workflows that are available for each individual Workgroup.
  • Status defines steps within a workflow.
  • Status Revised is an uneditable field that shows who updated the Change to the current status and when.
  • Type is the type of the Change, used to categorize and search for Change records.
  • Priority is the priority of the Change, used to categorize and search for Change records.
  • Complexity is the priority of the Change, used to categorize and search for Change records.
  • Submitted By is the Resource who suggested the Change.
  • Responsible is the Resource who is currently responsible for the Change at this status in the workflow.
  • Task Name is the Task that the Change is associated with. If a Change is associated with a specific Task, then the Change is displayed in the Time Tracking application so that time can be posted against the Change and rolled up into the Project Task. See Time Tracking.
  • Evaluation Date is the date that the Change was first suggested (which may be before it was actually entered into Project InVision).
  • Date Closed is the date the Change was actually completed (if the change was accepted) or the date the Change was deferred or refused.

Comments

This page is for free-form comments that are maintained with the Change record. See Detail Form Common Pages, Comments.

Keywords

This page allows you to define keywords for the Change. The keywords you can choose are defined by the Workgroup Administrator. See Detail Form Common Pages, Keywords.

Links

This page contains links to other records within Project InVision or to web addresses outside Project InVision. Links within Project InVision help you track associations between Projects and other items, such as Tasks and Resources. See Detail Form Common Pages, Links.

Discussions

This page contains threaded Discussions related to the Change. You can create a Discussion item, respond to the sender of a Discussion item, or respond to the group of users that are involved in the Discussion. See Detail Form Common Pages, Discussions.
