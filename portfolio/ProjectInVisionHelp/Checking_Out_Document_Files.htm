Checking Out Document Files

To make changes to a Document file, the file must be Checked Out of Project InVision. Checking out a file copies the most recent version of the file from Project InVision to your computer's hard disk, where you can make changes to the file. Once changes are made, the file can be Checked In to Project InVision so the changes are available to all users.

To check Out a Document file

  1. In a Documents list view, click the Document Title of a Document record that has a checked-in file.
    The Documents list can show either a subset of Documents, based on a selected Project or group of Projects, or all Documents. If you do not see the Document record you want to edit, make sure you have selected the Project that the Document record is part of, or select -All-.
  2. Click  Check Out.
    If the file is already checked out, you'll see the  Check In button instead.
    You can also click Check Out when you're viewing the Documents Detail Form.
  3. If you're asked whether to Open or Save the document, click Save, then navigate to the folder on your hard disk where you want to save the file and click Save.
    -or-
    If the file opens automatically in either Internet Explorer or its native application, save the file to your hard drive.

Because of the way files are transferred over the Web, spaces in the filename are replaced with the characters “%20” during check out. If you save the file as-is, those characters will remain in the filename and you'll have two similarly named files. To resolve this, review the File name field and replace any instances of %20 with a space by selecting the “%20” characters and pressing the spacebar. You can eliminate this as an issue by creating files with no spaces in the filenames.

Once you save the file to your hard drive, you can open and edit it. When you're finished editing, you should check the file back in.
