Synchronization

Each time an associated project schedule file (such as a Microsoft Project file) is checked in to a Project, an automatic bi-directional synchronization is performed between the associated project schedule file and the Project InVision database. This ensures that the data stored and displayed in Project InVision is accurate and up-to-date. Following is a discussion of the various functions that occur during the synchronization process:

If desired, you can disable the automatic synchronization functions described below on the Options page of the Projects Detail form

Information Synchronized

Auto-Synchronize of Project Schedule
  • Project Information (Project number, Title, Manager, etc.) is posted to the File, Properties area of the project schedule file. The file mapping is described below in the section Affected Project Schedule File Areas.
  • Resources defined as members of the Project Team in Project InVision are posted to the Resource Sheet in the project schedule file.
  • Detailed Task and Resource Assignment information from the project schedule file is imported and displayed in the Tasks module and Projects Detail Form.
Auto-Post Time
  • Actual Work hours and Remaining Work hours submitted in Time Tracking for Tasks are posted to the appropriate Task records in the project schedule file.
  • If they have been linked to a Task, Actual Work hours submitted in Time Tracking for Issues, Defects, Risks, Changes, and Requirements are rolled up into the appropriate Task records in the project schedule file.
Auto-Synchronize of Current Resource Requirements Plan
The Current Resource Plan is updated to reflect changes to Tasks and Resource Assignments based on the Project InVision Tasks module.
Auto-Synchronize of Calendar
Standard Working Hours and Non-Working Hours identified in Project InVision that occur in the duration of the Project are posted to the Standard Calendar in the project schedule file.
Auto-Synchronize of Cost Rates
The Internal Cost Rate from the Resource record, or Skill Category, in Project InVision is synchronized to Rate Table A of the project schedule file, updating the standard cost rate for the resource in the project schedule file.
Auto-Synchronize of Billing Rates
The Billing Rate from the Resource record, or Skill Category, in Project InVision is synchronized to Rate Table B of the project schedule file, updating the cost rate table for the resource in the project schedule file.

Affected Project InVision Areas

Once synchronization has occurred, information from the project schedule file is stored in the Project InVision database and displayed in various areas throughout Project InVision. Below is a description of the major areas in Project InVision that are affected by synchronization:

For a detailed mapping of all Project InVision fields that are affected by the synchronization process, see Appendix A – Field Mapping.

Projects Module Information:

Projects Pages Description
Information Data on the Information page is synchronized one way only: from Project InVision to the project schedule file, posted to the corresponding fields in the project schedule file. (See Affected Project Schedule File Areas.)
Comments Synchronization of the Comments page data may be configured to the requirements of each workgroup. The Workgroup Administrator can determine if Comments are exported from Project InVision to the project schedule file, imported from the project schedule file into Project InVision, or maintained separately.
Summary High-level view of the overall Project schedule showing Current, Baseline, and Actual Start and Finish Dates, Duration, Work, and Variances are imported from the project schedule file into Project InVision and displayed on the Summary page.
Key Deliverables Tasks or Milestones from the project schedule file where the “Marked” field is set to “Yes” will be imported from the project schedule file into Project InVision and displayed on the Key Deliverables page
Resources Resources who have been included on the Resource Sheet of the project schedule file will be imported into Project InVision and displayed on the Resources page. Resources that do not match resources already in the Project InVision database will be ignored.

Tasks Module Information:

Tasks Module Description
Information Detailed Task information, including Current, Baseline, and Actual Dates, Duration, Work, and Variances, will be imported from the project schedule file into the Project InVision Tasks module.
Comments Task Notes will be imported from the project schedule file into the Project InVision Tasks module and displayed on the Comments page.
Resources Detailed Resource assignments will be imported from the project schedule file into the Project InVision Tasks module and displayed on the Resources page.

Resources Module Information:

Resources Pages Description
Assignments Detailed Resource assignments will be imported from the project schedule file into Project InVision and displayed on the Assignments page.

Time Tracking Information:

Assignment Hierarchy View Description
Task Assignments Detailed Resource assignments will be imported from the project schedule file into the Project InVision Tasks module. These assignments will appear in the Assignments View of Time Tracking, where they can be selected and included in a Timesheet.
