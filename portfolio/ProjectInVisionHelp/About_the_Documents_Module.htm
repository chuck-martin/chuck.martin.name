About the Documents Module

The Project InVision Documents module is where you manage and collaborate on project-related documents. These Documents are usually project management work product, such as a project charter, specification, or test plan. In the Documents module, associated document files, such as Microsoft Word (.doc) files or files created by any other application, are stored and managed through the Project InVision security (check in and check out) and versioning features. You can view Document information in list views or in the Documents Detail Form.

Documents that are created during a Project can be stored (and versioned) within Project InVision and associated with those Projects. The Documents module captures detailed information about a Document record, including its Name, Type, Author, and Status. Documents can be tied to a Project Status such that once a Project reaches a particular Status, Document records may automatically be created for that Project record. These Document records then need to be updated and approved before the record can be advanced to the next Status.
