Projects Detail Form

The Projects Detail Form contains information about a Project, including the Title, the type of Project, the responsible Resources, Status, and Condition. The form contains several pages, each containing a specific type of information. Click the links under Pages to view each page. The form also has its own toolbar that allows quick access to common functions.

Project Detail Form Pages 

These are the default pages. Your Workgroup Administrator may have renamed existing pages or added new ones, or may have renamed, added, or deleted fields within any page.

Information

This page contains the basic Project information, including Project Number, Title, Project Type, Priority, Department, Region, Originator, Sponsor, Manager, Workflow, Status, Condition, Parent Project, Billing Code, Cost Center, and Request Date. Other useful information, such as the Template from which the Project Schedule was created, the project schedule file currently associated with the Project record, and the date and time that the Project schedule was Checked In to the system are displayed in a read-only format.

  • Project # is generated by Project InVision and is not editable. It is one way to identify the Project.
  • Project Title is a brief (up to 40 characters) description of the Project.
  • Type is the type of the Project, used to categorize and search for Project records.
  • Priority is the priority of the Project, used to categorize and search for Project records.
  • Department is the department responsible for the Project, used to categorize and search for Project records.
  • Region is the region where the Project is based, used to categorize and search for Project records.
  • Workflow defines the order in which the Status of a particular record may be changed in Project InVision. Each time the Status of a Project InVision record is changed, Project InVision logs the date, time, and User responsible for the change. Click the Status label to see Status History.
    The Workgroup Administrator defines the Workflows that are available for each individual Workgroup.
  • Status defines steps within a workflow.
  • Status Revised is an uneditable field that shows who updated the Project to the current status and when.
  • Condition is the condition of the Project, used to categorize and search for Project records.
  • Parent Project is used if the Project is a sub-Project of a larger Project, and identifies the larger Project.
  • Originator is the Resource who began the Project.
  • Sponsor is the Resource who is approved the Project.
  • Manager is the Resource who is responsible for the Project and defines where information and documents go in some workflows.
  • Template is an uneditable field that shows the template used to create the Project record.
  • Request Date is the date that the Project was first requested (which may be before it was actually entered into Project InVision.
  • Billing Code identifies what category Project costs belong to.
  • Cost Center identifies what department is responsible for paying Project costs.
  • Association shows the filename of the associated project schedule file.
  • Checked In/Checked Out shows the date and time that the associated project schedule file was checked in or checked out and who performed the action.

Summary

This page shows a high-level summary of project schedule information, including actual vs. baseline comparisons, to support the quick identification of variances. The information displayed in the Summary page is rolled up from the various Tasks captured in the Tasks module.

Key Deliverables

This page shows summary information about key dates in the progress of a project.

In the Tasks module, selecting the Key Deliverable check box on the Task record identifies it as a Key Deliverable. If the Project has an attached project schedule file, any task where the “Marked” field is set to “Yes” is regarded as a Key Deliverable by Project InVision and is displayed in Key Deliverable Views and Reports. For information on marking tasks, see your project management application documentation.

Resources

This page shows all Resources that have been defined as part of the Project Team. Resources that are assigned to the Project in Project InVision are also posted to the Resource Sheet of the associated project schedule file. Resources that have been assigned to the Project are available to be assigned to Tasks.

For more information, see Assigning Resources to Projects.

Keywords

This page allows you to define keywords for the project to assist in categorizing and filtering Project records. The keywords you can choose are defined by the Workgroup Administrator. See Detail Forms: Common Pages, Keywords.

Versions

This page allows you to view the versions of the project schedule file that have been checked in to the Project record and evaluate changes that have been made to the Project over time. New versions are created on Check In and a user with the proper permissions is able to mark specific Versions to be kept in the system.

Discussions

This page contains threaded Discussions related to the Project. You can create a Discussion topic, respond to the sender of a Discussion topic, or respond to the group of users that are involved in the Discussion. See Detail Forms: Common Pages, Discussions.

Project Charter

This page allows you to maintain information about the Charter for the Project, including the description, the project scope (vision), business case and benefit (purpose), and defined criteria to measure project success (success factors). Multiple factors can be entered, which allows Management to see the reasoning of why Projects were started.

Comments

This page is for free-form comments that are maintained with the project record. See Detail Forms: Common Pages, Comments.

Links

This page contains links to other records within Project InVision or to web addresses outside Project InVision. Links within Project InVision help you track associations between Projects and other items, such as Tasks and Resources. See Detail Forms: Common Pages, Links.

Options

This page is where you set options for how the Project is handled by Project InVision.

Costing

This page is where you can enter aggregate Current, Baseline, and Actual costing information for Resources, Materials, and Expenses.

Billing

This page is where you can enter and view aggregate billing information for Resources, Materials, and Expenses.

Scorecard

This page allows your organization to prioritize Projects based on a formula that gives different weightings to specific aspects of the Project. See Understanding the Project Scorecard
