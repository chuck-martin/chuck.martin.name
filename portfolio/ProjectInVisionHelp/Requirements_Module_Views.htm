Requirements Module Views

Default Views have been created for the Requirements module. Your security access determines what Views you can see.

Reports may be customized by the Workgroup Administrator or a Report Writer, so some View names or fields may not exist or may be labeled differently than shown here.

-Standard-

This View shows almost all the information in the Requirements Detail Form. Fields: Requirements Number, Requirements Title, Responsible, Submitted by, Status, Type, Priority, Complexity, Date Entered, Evaluation Date, Date Closed, Business Reason, Impact, Project Number, and Project Name.

Actual Work Pie

This View is a Pie Chart with slices based on Rollup selection and the Date Range specified. Depending on the Rollup used, the Work Hours will display for the specified Date Range, grouped by Rollup. Numbers can be rolled up by Type, Priority, Complexity, Status, Responsible, Submitted by, or Project. Fields: Requirement Number, Requirement Title, Status, Actual Work, and Timesheet Date.

Count Chart

This View shows a total count of Requirements separated into groups by Rollup. Numbers can be rolled up by Type, Priority, Complexity, Status, Responsible, Submitted by, or Project.

Detail

This View shows detailed information about Requirements that exist for selected Projects, grouped by Project. You can see information related to the Requirements, as well as who is responsible for it, when it was evaluated, and the current status of the Requirement. Fields: Project Number, Project Title, Requirement Number, Requirement Title, Responsible, Submitted by, Date Entered, Evaluation Date, Date Closed, Status, Priority, Complexity, Type, Description, Business Case, and Impact.

Personal Count Chart

This View shows a total count of Requirements, separated into groups by Rollup, and showing the number of Requirements per Rollup group where the current user is the Responsible party. Numbers can be rolled up by Type, Priority, Complexity, Status, Responsible, Submitted by, or Project.

Requirements Count Pie

This View is a pie chart that presents the pie slices based on Rollup selection. Depending on the rollup used, the counts will display grouped by the various values that exist for the Rollup field. Numbers can be rolled up by Type, Priority, Complexity, Status, Responsible, Submitted by, or Project. Fields,: Requirement Number, Requirement Title, Status, and Count.

Requirement Matrix

This View shows a matrix that compares Requirement Complexity to Requirement Priority for all Requirements selected. It separates the counts into High, Medium, and Low Complexity versus High, Medium, and Low Priorities.

Requirement Trend Chart

This View shows how Requirements have progressed over time.
