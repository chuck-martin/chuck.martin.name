About the Projects Module

The Project InVision Projects module is where you manage and collaborate on your entire portfolio of Projects. In the Projects module, you manage projects through a consistent lifecycle, from request through close-out, to assure their successful delivery with repeatable results. Project portfolio analysis tools, such as scorecards and bubble charts, help you to make better project prioritization decisions. You can view project information in list views or in the Projects Detail Form.

Projects are key records in Project InVision because all the other types of records are linked to a specific Project. A Project is specific to one Workgroup. The Projects module contains information about Projects; the information displayed depends on selections made in the Selector, View, Filter, and Group By options.

To create and maintain a Project in Project InVision, you should:

  1. Create the Project record in the Projects module .
  2. Add Resources to the Project to define the project team.
  3. Define a project task schedule by:
    • Creating and checking in a project schedule file, including tasks and estimates.
      -or-
    • Entering tasks directly in the Project InVision Tasks module.
      -or-
    • Selecting a Project template to create an associated project schedule file.
  4. Manage the Project, and its related elements (team, schedule, documents, issues, etc.) within the Portfolio Manager application.
